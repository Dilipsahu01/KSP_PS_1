# KSP "Command Center" UI/UX Blueprint

This document outlines the frontend architectural design for the Karnataka State Police (KSP) Crime Database AI Dashboard. Built on **Next.js (App Router)**, the UI is designed as a high-contrast, dark-mode "Command Center" specifically tailored for 24/7 law enforcement environments, eliminating cognitive overload while providing extreme analytical depth.

---

## 1. The "Omni-Search" Command Interface

The primary interaction zone must feel frictionless, fast, and unified. We abandon traditional form-based filtering for an intelligent, context-aware command bar.

### Visual Layout
*   **Position:** Fixed at the bottom-center of the screen (similar to ChatGPT) or top-center (like Mac Spotlight), enclosed in a floating, frosted-glass container (`backdrop-blur-md`).
*   **Input Mechanisms:** A wide, expanding text area alongside a prominent microphone action button.
*   **Language Toggle:** A subtle floating pill above the input to toggle between `[ EN | ಕನ್ನಡ ]`.

### Voice Interaction (WebRTC / MediaRecorder)
*   When the microphone is tapped, the UI transitions to a "Listening" state.
*   **Visualizer:** Implement a CSS/Canvas audio wave visualizer bound to the microphone's decibel output to provide immediate hardware feedback.
*   **Streaming Text:** As Zoho Zia/Bhashini translates the audio, stream the text into the input box in real-time so the officer can verify the transcription before submission.

### The "Agentic" Waiting State
Instead of a generic spinning loader, we expose the backend routing process to build trust.
*   Render a subtle step-by-step progress indicator (Timeline) above the search bar based on SSE (Server-Sent Events) or WebSockets from the backend:
    *   *Step 1:* `Analyzing Intent (Semantic Router)...`
    *   *Step 2:* `Intent matched: TEXT_TO_SQL. Generating query...`
    *   *Step 3:* `Executing query against District 12 dataset...`

## 2. The Three Dynamic View Modes

The backend returns an `intent_routed` flag (`TEXT_TO_SQL`, `RAG`, or `GRAPH`). The React UI uses a dynamic rendering switch to swap out the main content area based on this flag.

### A. The Analytical View (Structured Data / SQL)
*   **Objective:** Display quantitative trends and forecasts.
*   **Layout:** A multi-pane dashboard. Top row contains "KPI Metric Cards" (e.g., Total Crimes, Clearance Rate). The main area displays charts.
*   **Library Recommendation:** **Tremor** (tremor.so). Built on Tailwind CSS, Tremor is the industry standard for React analytical dashboards. It provides drop-in components for highly polished Bar Charts, Donut Charts, and Heatmaps that fit a command-center aesthetic perfectly.

### B. The Investigative View (Unstructured RAG)
*   **Objective:** Display case facts, modus operandi summaries, and concrete proof.
*   **Layout:** A split-screen layout (60/40).
    *   **Left Pane (60%):** The NLP Summary generated by the LLM, formatted nicely with Markdown (lists, bolded entities).
    *   **Right Pane (40%):** A scrollable "Source Evidence" feed. This renders the `citations` array from the JSON payload. Each citation is a stylized Card showing the source PDF name, page number, and the exact highlighted text snippet the AI used.

### C. The CID Network View (Knowledge Graph)
*   **Objective:** Visually map criminal syndicates, accomplices, and financial ties.
*   **Layout:** A full-screen interactive canvas.
*   **Library Recommendation:** **react-cytoscapejs**. While React Flow is great for process diagrams, Cytoscape was built for dense biological and network topologies. It natively handles complex physics layouts (force-directed, circular), enabling analysts to drag, zoom, and expand nodes (Accused -> Case -> Complainant).

## 3. The Trust & Evidence Panel (Explainable AI)

Law enforcement decisions must stand up in court. The AI cannot be a black box.

*   **The Component:** A global right-side slide-out drawer (using the `Sheet` component from `shadcn/ui`).
*   **The Trigger:** Every AI response includes a highly visible "View Reasoning & Evidence" button with a shield or code icon.
*   **Drawer Contents:**
    1.  **Reasoning Path:** A vertical stepper (`Timeline` component) showing the exact logical steps the AI took (e.g., "Filtered by District 12", "Found 4 similar historical cases").
    2.  **Raw Execution Details:** For SQL queries, display the raw executed MySQL string inside a Syntax Highlighter block (using `react-syntax-highlighter` with a dark theme). Include a "Copy Query" button for analysts who want to run it manually.
    3.  **Confidence Score:** A visual gauge indicating the retrieval confidence score.

## 4. Component Architecture & State Management

### Framework & Styling
*   **Framework:** Next.js App Router (`app/`).
*   **Styling:** Tailwind CSS with a strict CSS Variables theme.
*   **Color Palette:** Deep navy/slate backgrounds (`bg-slate-950`), neon cyan/blue primary accents (hacker/tech aesthetic), and amber/red alerts for critical data.

### UI Library Selection
*   **shadcn/ui:** The absolute best choice for this project. It is not a rigid npm package; it provides copy-paste, highly accessible Radix UI primitives styled with Tailwind. We will use it for Buttons, Cards, Data Tables, Dialogs, and Sheets.
*   **Lucide React:** For crisp, modern iconography.

### State Management
*   **Zustand:** Avoid Redux. Use Zustand for lightweight, global state management. 
*   **Why?** When the Omni-Search bar (in the footer) receives the massive XAI JSON payload from the API, it needs to instantly update the global state. The Dynamic View component (in the center) and the Trust Panel (in the right drawer) will instantly re-render by subscribing to this single Zustand store.

### Next.js File Structure

```text
/app
  /(dashboard)
    layout.tsx               // Sidebar nav, global header, layout wrappers
    page.tsx                 // The main view switcher (renders SQL/RAG/Graph based on state)
  /api                       // (Backend orchestrator from Phase 3)
/components
  /omni-search               // Microphone, input, animation states
  /views
    AnalyticalView.tsx       // Tremor charts
    InvestigativeView.tsx    // RAG summary + citations
    NetworkView.tsx          // Cytoscape canvas
  /explainability
    TrustDrawer.tsx          // Slide-out XAI panel
/store
  useCommandStore.ts         // Zustand state: { currentPayload, isLoading, currentView }
```
